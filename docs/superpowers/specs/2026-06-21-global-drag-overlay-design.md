# macOS 全局拖拽渐变框设计规范

日期：2026-06-21  
应用工作名：DragFrame

## 1. 目标

开发一个 macOS 原生菜单栏工具。用户按住已配置的修饰键组合并使用鼠标左键拖动时，工具在所有应用上方显示一个圆角渐变描边方框；鼠标松开或快捷键被释放后，方框立即消失。

工具只提供视觉反馈，不截图、不记录选区、不改变鼠标事件，也不执行任何选中区域操作。

## 2. 已确认的产品行为

- 应用常驻菜单栏，不显示 Dock 图标。
- 默认触发组合为 `Control + Option（⌃⌥）`。
- 用户可以在设置中重新绑定修饰键组合。
- 重绑定仅接受 `Command`、`Option`、`Control`、`Shift` 的组合，不接受字母、数字或功能键。
- 仅鼠标左键拖动触发；普通点击、右键和滚轮不触发。
- 鼠标移动超过 3pt 后才显示方框，避免点击抖动造成闪烁。
- 鼠标松开，或拖动过程中释放任一必需修饰键，方框立即消失。
- 覆盖框忽略鼠标事件，原应用仍能正常接收拖文件、文字选择和窗口操作。
- 支持从任意方向拖动，也支持多显示器和包含负坐标的显示器布局。

## 3. 视觉规范

- 内部完全透明，不对框内内容做任何染色、模糊或遮挡。
- 描边宽度为 3pt。
- 圆角半径为 18pt；尺寸过小时将半径限制为短边的一半，避免路径畸变。
- 描边使用暖色到冷色的渐变：橙黄 → 粉红 → 紫色。
- 使用轻量外阴影增强复杂背景上的可见性，不使用强烈霓虹光晕。
- 方框跟随当前鼠标位置实时改变大小，动画不得造成明显滞后。

## 4. 技术架构

应用使用 Swift 编写，采用 AppKit 与 SwiftUI 混合架构：

1. `AppDelegate` 管理应用生命周期、菜单栏和设置窗口。
2. `GlobalEventMonitor` 使用只读 `CGEventTap` 观察左键按下、左键拖动、左键松开和修饰键变化。回调始终返回原事件，不拦截或修改事件。
3. `DragStateMachine` 将原始事件转换为 `idle`、`pressed`、`dragging` 三种状态，负责快捷键匹配、3pt 阈值、起点与当前点管理，以及取消逻辑。
4. `OverlayWindowController` 管理一个不激活、透明、无标题栏、忽略鼠标的 `NSPanel`。窗口可加入所有 Space，并作为全屏辅助窗口显示在全屏应用上方。
5. `GradientBorderView` 使用 Core Animation 图层绘制透明内部、渐变描边、圆角和轻阴影。
6. `ShortcutSettings` 使用 `UserDefaults` 保存修饰键掩码，并向事件状态机提供当前设置。
7. SwiftUI 设置窗口提供修饰键选择、恢复默认值和权限状态展示。

模块之间只传递拖拽状态、屏幕坐标矩形和设置值；事件监听层不直接操作视图，绘制层不读取全局事件。

## 5. 事件与坐标流程

1. 左键按下时，若当前修饰键包含已配置的组合，记录全局起点并进入 `pressed`；否则保持 `idle`。额外按下未配置的修饰键不影响匹配。
2. 接收到左键拖动后计算起点与当前点的距离。未超过 3pt 时不创建可见框。
3. 超过阈值后进入 `dragging`，把 Quartz 全局坐标转换为 AppKit 全局坐标，生成标准化矩形并更新覆盖窗口。
4. 用户可以向左上、右上、左下或右下拖动；标准化矩形始终具有非负宽高。
5. 覆盖窗口可以跨越显示器边界；显示器排列造成的负坐标必须保留。
6. 左键松开、必需修饰键被释放、监听权限丢失或事件监听停止时，状态返回 `idle` 并隐藏窗口。

如果拖动开始后用户额外按下其他修饰键，拖拽继续；只有已配置组合中的必需修饰键被释放时才取消。

## 6. 菜单栏与设置

菜单栏菜单包含：

- “启用 DragFrame”开关；默认启用。
- “设置…”：打开 SwiftUI 设置窗口。
- 当前快捷键的只读展示。
- 权限缺失时显示“需要输入监控权限”和“打开系统设置”。
- “退出 DragFrame”。

设置窗口包含四个修饰键复选项、当前组合预览和“恢复默认值”按钮。至少必须选择一个修饰键；尝试清空全部选择时，界面保持上一个有效组合并给出内联说明。新组合保存后立即生效，应用重启后继续使用。

首版不包含开机自启动、颜色自定义、描边尺寸自定义、截图、坐标显示和历史记录。

## 7. 权限与异常处理

- 首次运行时先解释用途，再通过系统 API 请求监听输入事件所需的隐私权限。
- 未授权时不启动全局监听；菜单栏图标与菜单显示警示状态，并提供打开对应系统设置页面的入口。
- 用户从系统设置返回后，应用在重新激活和定时轻量检查时更新权限状态，无需重启应用。
- 如果事件 tap 因超时或用户输入被系统临时禁用，应用尝试重新启用。
- 如果重新启用失败或权限被撤销，立即隐藏覆盖框、停止拖拽状态并展示权限提示。
- 所有 UI 更新调度到主线程；事件 tap 回调只做轻量状态转发，避免系统因回调超时禁用监听。

## 8. 验收与测试

### 单元测试

- 快捷键匹配：默认组合、自定义组合、额外修饰键和缺少必需修饰键。
- 状态机：按下、阈值内移动、超过阈值、拖动、松开和快捷键释放。
- 矩形标准化：四个拖动方向、零尺寸和负坐标。
- 设置持久化：有效组合保存、恢复默认值和空组合拒绝。

### 手工验收

- 在 Finder、浏览器、文本编辑器和桌面中拖动，方框显示且不影响原操作。
- 方框内部完全透明，3pt 渐变描边、18pt 圆角和轻阴影符合确认的 B 方案。
- 点击但不拖动时不显示；松开鼠标或释放快捷键时立即消失。
- 横跨两台显示器以及从负坐标显示器开始拖动时，方框位置正确且连续。
- 普通拖拽、右键、滚轮和未按快捷键的左键拖动不触发。
- 修改快捷键后立即生效，重启应用后设置仍保留。
- 未授权、撤销权限和事件 tap 被禁用时，不残留覆盖窗口且提示可恢复。
- 全屏应用与不同 Space 中的可见性按设计工作。

## 9. 完成标准

工程能够在当前 Xcode 稳定版中构建和运行；单元测试通过；上述手工验收项无阻塞问题。交付内容包括 Swift/Xcode 工程、测试、简短 README，以及从 Xcode 运行和授予权限的说明。
