# First Launch Onboarding Design

## 背景

DragFrame 的核心功能依赖 macOS 输入监控权限。现在首次启动时会弹出一个系统 alert 解释权限，再进入设置/权限恢复流程。这个方式能工作，但不够像一个成熟的原生 macOS 工具：信息密度偏高，缺少产品欢迎感，也没有清楚告诉用户授权后如何试用。

本设计将首次启动体验升级为轻量欢迎与授权引导窗口。

## 产品目标

- 首次启动且缺少输入监控权限时，展示一个干净的欢迎窗口。
- 用简短文案说明 DragFrame 的用途和为什么需要输入监控权限。
- 提供明确主按钮打开输入监控设置。
- 授权后提示用户按住当前快捷键并拖动鼠标试用。
- 已授权用户不自动打扰。
- 设置页保留“打开欢迎引导”入口，便于用户重新查看。

## 非目标

本轮不做：

- 多页复杂教程。
- 视频或动画演示。
- 完整交互式沙盒试用区域。
- 修改快捷键设置方式。
- 修改拖拽、事件吞噬、颜色设置或登录项逻辑。

## 用户体验设计

新增一个原生欢迎窗口，标题为“欢迎使用 DragFrame”。

窗口内容分为三个简洁区块：

1. “框选屏幕重点”
   - 说明：按住快捷键并拖动鼠标左键，即可在任何应用上方显示透明渐变方框。
2. “需要输入监控权限”
   - 说明：DragFrame 只读取修饰键和鼠标拖动，用来显示方框和避免底层误选，不保存输入内容。
3. “试一下”
   - 已授权时：显示当前快捷键，例如“按住 ⇧⌥ 并拖动鼠标左键试试看”。
   - 未授权时：提示先授权，再回到任意应用试用。

主要按钮：

- 未授权：`打开输入监控设置`
- 已授权：`开始使用`

次要按钮：

- `稍后`

设置页权限分组增加一个小按钮：

- `打开欢迎引导`

## 展示规则

- 如果输入监控权限已授予，启动时不自动显示欢迎窗口。
- 如果输入监控权限缺失，并且本机还没有展示过欢迎窗口，则启动后自动显示欢迎窗口。
- 欢迎窗口展示过一次后，不再因为定时权限刷新反复弹出。
- 如果用户之后权限缺失或监听失败，仍然使用现有设置/恢复窗口提示。
- 用户可从设置页手动重新打开欢迎窗口。

## 架构设计

### `WelcomeWindowController`

新增 AppKit window controller：

- 承载 SwiftUI `WelcomeView`
- 管理窗口尺寸、标题、居中和激活
- 可重复 present，但同一时间只保留一个窗口

### `WelcomeView`

新增 SwiftUI view：

- 观察 `InputMonitoringPermission`
- 读取当前 `ModifierShortcut`
- 展示欢迎、权限说明和试用提示
- 调用传入的 `openPrivacySettings`
- 调用传入的 `dismiss`

### `AppDelegate`

替换旧的 alert-based `presentPermissionExplanationIfNeeded()`：

- 使用新的 UserDefaults key：`dragFrame.didShowWelcomeGuide`
- 启动时如果缺少权限且未展示过欢迎窗口，则展示欢迎窗口并记录 key
- 如果已展示过欢迎窗口但仍缺少权限，则沿用现有恢复窗口
- 为设置页提供 `presentWelcomeGuide()` 回调

### `SettingsWindowController` / `SettingsView`

增加 `openWelcomeGuide` 回调，并在权限分组中显示“打开欢迎引导”按钮。

## 错误处理

- `CGRequestListenEventAccess()` 返回 false 时，继续调用系统设置 URL。
- 欢迎窗口关闭不影响主应用运行。
- 权限状态变化后，欢迎窗口内容通过 `InputMonitoringPermission` 自动刷新。

## 测试策略

### 自动测试

运行现有 XCTest，确保核心拖拽逻辑不回归。

### 手动验证

- 重置输入监控权限后首次启动，显示欢迎窗口。
- 点击“打开输入监控设置”能进入系统设置。
- 授权后欢迎窗口显示“开始使用”和试用提示。
- 已授权启动时不自动弹欢迎窗口。
- 设置页可以重新打开欢迎引导。
- 拖拽框、颜色设置、登录项、菜单栏状态不回归。

## 验收标准

- 首次缺少权限时不再弹出旧的系统 alert，而是显示欢迎窗口。
- 欢迎窗口文案简洁，符合原生 macOS 工具气质。
- 设置页能重新打开欢迎引导。
- 权限恢复逻辑仍然有效。
- 测试通过，README、开发文档和 roadmap 更新。
