# DragFrame Native Product Polish Design

## 背景

DragFrame 已经具备核心能力：按住快捷键并拖动鼠标左键时，在所有 macOS 应用上方显示内部透明的渐变圆角方框，并避免底层应用误选中文本。下一阶段目标不是增加复杂功能，而是把它打磨成一个更成熟、更有苹果原生气质的菜单栏工具。

## 产品目标

这一轮优化追求“小而精”：

- 首次启动时，用户能快速理解 DragFrame 的用途和权限要求。
- 菜单栏状态清晰，用户能知道应用是已就绪、缺少权限、监听失败，还是已暂停。
- 设置窗口更像原生 macOS 工具：克制、干净、少解释、多状态。
- 支持登录时自动启动，符合常驻菜单栏工具的使用习惯。
- Overlay 样式在代码层集中管理，为后续样式设置打基础，但本轮不开放复杂主题面板。

## 非目标

本轮不做以下能力：

- 多套用户可选颜色主题。
- 拖拽尺寸标签。
- 复杂快捷键录制器。
- 截图、保存、标注、历史记录。
- 云同步或账号系统。

## 用户体验设计

### 首次启动与权限恢复

设置窗口将承担轻量引导职责。它应该显示：

1. DragFrame 当前是否已启用。
2. 输入监控权限是否已授予。
3. 全局监听是否正常运行。
4. 当权限缺失或监听失败时，给出明确的恢复动作。

文案风格应简短直接，避免技术术语堆叠。权限异常时，优先给出“打开输入监控设置”；监听失败且权限显示正常时，提示用户可能需要重置授权记录并重新打开应用。

### 菜单栏状态

菜单栏图标继续使用 template SF Symbol，让 macOS 自动适配浅色和深色状态栏。不同状态通过菜单项和 tooltip 表达：

- 已就绪：DragFrame 已就绪。
- 已暂停：DragFrame 已暂停。
- 缺少权限：需要输入监控权限。
- 监听失败：输入监听未启动。

菜单中保留最少必要入口：

- 启用或暂停 DragFrame。
- 当前快捷键。
- 状态/权限提示。
- 打开设置。
- 打开系统设置。
- 退出。

### 设置窗口

设置窗口保持单页结构，但视觉上更接近 macOS 原生偏好设置：

- 顶部显示产品名、用途和当前状态。
- “快捷键”分组保留 `Option`、`Shift`、`Command` 勾选配置。
- “启动”分组增加“登录时启动 DragFrame”。
- “权限”分组显示输入监控状态和恢复按钮。
- “外观”暂时只展示当前边框风格说明，不开放复杂配置。

### 登录时自动启动

使用 macOS 原生 `ServiceManagement` 登录项能力。设置项即时生效：

- 打开时注册登录项。
- 关闭时取消注册。
- 读取失败或注册失败时在设置页显示简短错误。

最低系统版本保持 macOS 13，因此可以使用 `SMAppService.mainApp`。

### Overlay 样式配置化

新增 `OverlayStyle`，集中管理：

- 边框宽度。
- 圆角半径。
- 内容外扩 inset。
- 渐变颜色、位置和方向。

当前默认样式保持现有视觉：6pt 粗边框，18pt 圆角，橙黄到粉紫渐变，框内完全透明，无阴影、无遮罩、无背景色。

## 架构设计

### 新增或调整模块

- `OverlayStyle`：AppKit 层样式模型，供 `GradientBorderView` 使用。
- `LaunchAtLoginController`：封装 `SMAppService.mainApp` 的读取、注册和取消注册。
- `RuntimeStatus`：继续作为运行状态来源，可扩展表达 paused、ready、permissionMissing、monitorFailed。
- `SettingsView`：重组 UI 文案和分组，减少散落状态判断。
- `StatusItemController`：根据启用状态、权限状态和监听状态更新菜单栏图标、tooltip 和菜单提示。

### 数据流

1. `AppDelegate` 初始化权限、运行状态、登录项控制器、菜单栏和设置窗口。
2. `InputMonitoringPermission` 定期刷新权限。
3. `DragCoordinator` 启停全局事件监听。
4. `RuntimeStatus` 记录监听失败或恢复。
5. `StatusItemController` 和 `SettingsView` 观察这些状态并展示给用户。
6. `LaunchAtLoginController` 只处理登录项，不参与拖拽逻辑。

## 错误处理

- 输入监控权限缺失：显示权限提示和“打开系统设置”。
- 权限看似存在但监听失败：提示可能需要重置授权记录。
- 登录项注册失败：保留当前值，显示失败信息，不影响 DragFrame 主功能。
- Overlay 样式配置错误：本轮不提供用户输入，因此使用固定默认值，不产生运行时错误。

## 测试策略

### 单元测试

- `OverlayStyle.default` 的关键参数稳定。
- 登录项控制器的状态转换通过可注入 client 测试。
- `RuntimeStatus` 的状态文案和优先级稳定。

### 手动验证

- 首次启动无权限时，设置页和菜单栏提示正确。
- 授权后，菜单栏显示已就绪，快捷键拖拽正常。
- 暂停 DragFrame 后，拖拽不显示方框，也不吞事件。
- 开启/关闭登录启动时，设置页没有卡顿或错误。
- 深色/浅色状态栏中，菜单栏图标保持清晰可见。

## 验收标准

- 应用仍然保持菜单栏常驻、无 Dock 图标。
- 默认拖拽行为不变。
- 框内仍完全透明，不覆盖底层内容。
- 设置窗口更简洁，状态更明确。
- 登录时启动可以在设置中打开和关闭。
- 单元测试通过，README 和开发文档更新。
