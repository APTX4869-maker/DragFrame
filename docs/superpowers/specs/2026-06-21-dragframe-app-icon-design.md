# DragFrame macOS 应用图标设计规范

日期：2026-06-21

## 目标

为 DragFrame 设计一枚符合 macOS 原生应用气质的图标，清晰表达“按住快捷键拖动鼠标，显示渐变圆角选区框”的功能，并替换当前默认应用图标。

## 视觉方向

采用用户确认的“白色轻量工具图标”方案：

- 暖白色 macOS 圆角图标底座，使用极淡灰色边缘和柔和阴影。
- 中央主体为四个粗圆角角标组成的选区框。
- 角标使用与应用一致的橙色 → 粉红 → 紫色渐变。
- 深色简洁鼠标指针融入选区框右下角，表达点击拖动。
- 参考 Telegram 与 ToDesk 图标的克制程度、留白和小尺寸辨识度，但不复制其图形或品牌元素。
- 正视、居中、平衡构图，在 16×16 小尺寸下仍能辨认选区框和指针。
- 不包含文字、字母、快捷键符号、截图内容、品牌标识或水印。

## GPT Image 2 生成要求

- 用例：`logo-brand`
- 输出：1024×1024 方形高质量主图。
- 风格：现代 macOS 原生应用图标、轻量、简洁、低拟物感。
- 保持明确轮廓和充足安全边距，不让主体触碰画布边缘。
- 避免高饱和整面背景、厚重金属边框、过强霓虹、复杂纹理、细碎装饰和拟真桌面场景。

## 工程接入

1. 先生成预览图，只在对话中展示，不修改工程或已安装应用。
2. 用户明确确认预览图后，保存 1024×1024 主图到项目资源目录。
3. 建立 `Assets.xcassets/AppIcon.appiconset`。
4. 从主图生成 macOS 所需的 16、32、128、256、512 和 1024 像素 PNG 文件。
5. 填写 `Contents.json`，设置 Xcode 的 `ASSETCATALOG_COMPILER_APPICON_NAME` 为 `AppIcon`。
6. 重新生成 Xcode 工程、构建 Release 版本并安装到 `/Applications/DragFrame.app`。

## 验收标准

- Finder、应用程序目录和系统相关界面不再显示默认图标。
- 1024×1024 主图清晰，无文字、乱码、水印或异常边缘。
- 16×16 和 32×32 尺寸仍能识别渐变选区框与鼠标指针。
- 图标视觉与应用实际的透明渐变框功能一致。
- 未获得用户对预览图的明确确认前，不替换任何应用图标资源。
- Xcode 构建成功，已有单元测试继续通过。
