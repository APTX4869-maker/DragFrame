# DragFrame 快捷键与覆盖框修复设计

日期：2026-06-21

> 说明：本文记录的是当时的修复设计。当前实现后来引入 active event tap，只在快捷键框选期间接管左键拖拽，避免底层内容被选中。当前行为以 `docs/development.md` 为准。

## 问题

1. 默认快捷键包含 `Control`。macOS 会把 `Control + 左键`解释成辅助点击，因此拖动开始时可能弹出右键菜单。
2. 当前渐变描边下面还有紫色阴影层，阴影会向框内扩散，产生类似背景色覆盖的视觉效果；3pt 描边也不够醒目。

## 快捷键修复

- 默认快捷键从 `Control + Option（⌃⌥）`改为 `Shift + Option（⇧⌥）`。
- 当前最终实现使用 active `CGEventTap`，只在快捷键框选期间接管这一段左键拖拽；非快捷键鼠标事件继续传给原应用。
- `Control` 不再是可绑定的触发修饰键；设置界面仅显示 `Option`、`Shift` 和 `Command`。
- 启动时检查已保存组合。任何包含 `Control` 的旧设置都迁移为新默认组合 `⇧⌥`，并立即持久化。
- 菜单栏和设置窗口同步展示迁移后的组合。

不采用保留 `Control` 并修改鼠标事件的方案，因为该方案需要可写事件 tap 和更高权限，也会违背“不干扰原应用操作”的设计目标。

## 覆盖框修复

- `NSPanel`、内容视图和所有绘制图层均保持透明背景。
- 删除独立的紫色描边阴影层，不绘制任何填充、染色、模糊或内阴影。
- 选区内部 100% 透明，底下原始屏幕内容不发生颜色变化。
- 渐变描边从 3pt 加粗为 6pt。
- 描边颜色继续使用橙黄 → 粉红 → 紫色渐变。
- 圆角半径保持 18pt；小尺寸矩形仍按短边的一半限制半径。
- 覆盖窗口仍忽略鼠标事件，松开鼠标或快捷键后立即隐藏。

## 测试与验收

- 默认快捷键显示为 `⇧⌥`。
- 新安装和无设置状态使用 `⇧⌥`。
- 保存的 `⌃⌥`、`⌃⇧` 或任何其他含 `Control` 的组合会迁移为 `⇧⌥`。
- 不含 `Control` 的有效自定义组合保持不变。
- 设置界面无法重新选择 `Control`。
- 使用默认快捷键拖动时不再出现右键菜单。
- 框内像素视觉上不发生颜色覆盖；仅显示 6pt 渐变圆角描边。
- 单元测试、Debug 测试和 Release 构建通过，安装版可启动并建立全局事件 tap。
