# Overlay Fade Out Design

## 背景

DragFrame 当前在鼠标松开后会立即隐藏覆盖框。功能正确，但视觉上略显突然。用户希望方框消失时有一个渐进消失的效果，让工具更有原生 macOS 应用的细腻感。

## 产品目标

- 鼠标松开后，覆盖框平滑淡出，而不是突然消失。
- 显示和拖动过程继续保持立即响应、精准跟手。
- 淡出过程中如果用户开始新的拖拽，立即取消淡出并显示新方框。
- 不改变拖拽状态机、事件监听、事件吞噬、权限、颜色和菜单栏逻辑。

## 非目标

本轮不做：

- 缩放动画。
- 弹性动画。
- 发光、阴影、模糊或填充效果。
- 用户可配置动画时长。
- 对显示过程加动画。

## 交互设计

覆盖框隐藏时执行轻量淡出：

- 初始透明度：`1.0`
- 结束透明度：`0.0`
- 动画时长：`0.16s`
- 动画完成后再真正 `orderOut`
- 下一次 `show()` 时立即恢复 `alphaValue = 1.0`

如果淡出过程中收到新的 `show(selectionRect:)`：

1. 取消旧隐藏结果。
2. 恢复窗口透明度为 `1.0`。
3. 更新 frame 并继续显示新方框。

## 架构设计

只修改 `OverlayWindowController`。

新增状态：

- `fadeOutDuration`
- `animationGeneration`

`animationGeneration` 用于避免旧动画 completion 在新拖拽开始后错误地隐藏窗口。

## 错误处理

- 如果窗口当前不可见，`hide()` 不启动动画，只确保 `alphaValue = 1.0`。
- 如果动画完成前新方框显示，旧 completion 不再执行 `orderOut`。
- 如果多次调用 `hide()`，以最后一次隐藏动画为准。

## 测试策略

### 自动测试

运行现有 XCTest，确保拖拽状态机和事件接管逻辑不回归。

### 手动验证

- 按住快捷键拖拽后松开，方框平滑淡出。
- 淡出时间短，不拖泥带水。
- 淡出过程中再次拖拽，新方框立即出现，不闪烁。
- 框内仍完全透明。

## 验收标准

- 松开鼠标时覆盖框渐进消失。
- 显示和拖动仍然即时响应。
- 新拖拽能打断旧淡出。
- 测试通过，应用重新安装并推送。
