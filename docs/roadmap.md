# DragFrame Roadmap

这个文档记录 DragFrame 后续值得优化的方向。原则：保持工具轻、快、克制，优先做能提升日常使用体验和稳定性的功能。

## P0：高优先级

### 自定义方框颜色（已完成第一版）

允许用户调整方框边框颜色，同时保持框内完全透明。第一版已实现少量预设和自定义三色渐变；后续如有需要，再考虑更多预设或导入导出。

### 首次启动欢迎与授权引导（已完成第一版）

增加轻量欢迎流程：介绍用途、引导输入监控授权、提示用户试用当前快捷键拖拽。第一版已实现欢迎窗口、打开输入监控设置、授权后试用提示，以及设置页重新打开入口。

### 事件监听自动恢复

当 event tap 被系统临时禁用或启动失败时，优先自动恢复；连续失败后再提示用户，避免过早打扰。

### 快捷键录制体验

将当前 `Option / Shift / Command` 勾选方式升级为“录制快捷键”：用户点击按钮后直接按组合键，应用自动识别并保存。继续拒绝 `Control`，避免触发 macOS Control-click 右键行为。

## P1：中优先级

### 多显示器和全屏场景深度测试

重点验证副屏在主屏左侧/上方、全屏 Space、Stage Manager、Retina 与非 Retina 混用等场景。

### 设置页试用区域

在设置页中增加小型预览或试用提示，让用户能感知当前边框样式，而不是只阅读说明文字。

### 登录项异常恢复

当 `SMAppService.mainApp` 状态异常或注册失败时，提供“打开登录项设置”的入口和更明确的恢复说明。

## P2：发布体验

### 正式分发

如果 DragFrame 要给其他用户安装，需要补齐：

- Developer ID 签名
- Notarization 公证
- `.dmg` 安装包
- GitHub Release
- 自动更新能力，例如 Sparkle
